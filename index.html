The projectDisruption Feed

We're trying to improve rail disruption APIs, and build visualisations for customers.

Data entry point

The data is currently wrote for people. APIs are for machines. Historically, developers have relied on unreliable hacks to build systems around it.

We're attacking the root of this problem - the data entry point. We aim to help build a backwards-compatible system that has structured data that's easy for developers to use.

Visualisations

Using structured disruption data we build visualisations for passangers, personalised calendar feeds, and even tools for members of the staff.
